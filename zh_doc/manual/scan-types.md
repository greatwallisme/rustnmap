# RustNmap 扫描类型

> **版本**: 1.0.0
> **状态**: 此文档描述 RustNmap 1.0.0 的扫描类型。2.0 版本开发中，详见 [CHANGELOG.md](../CHANGELOG.md)。

> 每种扫描类型的详细文档

---

## 概述

RustNmap 支持 12 种不同的扫描类型，每种都为特定场景设计。了解每种扫描的工作原理有助于您为目标环境选择正确的技术。

---

## TCP SYN 扫描 (`-sS`)

### 描述

TCP SYN 扫描是默认且最受欢迎的扫描类型。它通过发送 SYN 数据包并分析响应来执行"半开"扫描，而不完成 TCP 三次握手。

### 工作原理

```
Scanner          Target
   |    SYN      |
   | ----------> |
   |  SYN-ACK    |  <- 端口开放
   | <---------- |
   |    RST      |  <- 发送 RST 而非 ACK
   | ----------> |

   |    SYN      |
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |

   |    SYN      |
   | ----------> |
   |   (timeout) |  <- 端口被过滤
   |             |
```

### 用法

```bash
# 默认扫描
sudo rustnmap -sS 192.168.1.1

# 指定端口
sudo rustnmap -sS -p 22,80,443 192.168.1.1

# 带服务检测
sudo rustnmap -sS -sV 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 速度 | 快 |
| 可靠性 | 高 |
| 防火墙友好 | 中等 |

### 优点

1. **隐秘**：不完成 TCP 连接，不太可能被记录
2. **快速**：无需完成三次握手
3. **可靠**：对大多数 TCP 协议栈有效

### 缺点

1. 需要 root/管理员权限
2. 仍可能被现代 IDS/IPS 系统检测

---

## TCP Connect 扫描 (`-sT`)

### 描述

TCP Connect 扫描执行完整的三次 TCP 握手。当 SYN 扫描不可用时（无 root 权限），这是默认选项。

### 工作原理

```
Scanner          Target
   |    SYN      |
   | ----------> |
   |  SYN-ACK    |  <- 端口开放
   | <---------- |
   |    ACK      |  <- 完成握手
   | ----------> |
   |   (data)    |
   | <---------> |
   |  RST/FIN    |  <- 关闭连接
   | ----------> |

   |    SYN      |
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |
```

### 用法

```bash
# 无 root 权限
rustnmap -sT 192.168.1.1

# 全端口扫描
rustnmap -sT -p- 192.168.1.1

# 带操作系统检测
rustnmap -sT -O 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 否 |
| 隐秘 | 否 |
| 速度 | 中等 |
| 可靠性 | 高 |
| 被记录 | 是 |

### 优点

1. **无需 root**：使用标准用户权限即可工作
2. **可靠**：标准 TCP 连接
3. **通用**：在所有系统上工作

### 缺点

1. **被记录**：完整连接会被目标记录
2. **较慢**：必须完成完整握手
3. **资源密集**：在目标上使用更多资源

## UDP 扫描 (`-sU`)

### 描述

UDP 扫描通过发送 UDP 数据包并分析响应（或缺乏响应）来检测开放的 UDP 端口。由于 UDP 的无连接特性，UDP 扫描通常比 TCP 扫描慢。

### 工作原理

```
Scanner          Target
   |   UDP       |
   | ----------> |
   |   UDP       |  <- 端口开放（应用响应）
   | <---------- |

   |   UDP       |
   | ----------> |
   |   ICMP      |  <- 端口关闭（ICMP 端口不可达）
   | <---------- |
   |  Unreachable|

   |   UDP       |
   | ----------> |
   |  (timeout)  |  <- 端口开放或被过滤
   |             |
```

### 用法

```bash
# UDP 扫描
sudo rustnmap -sU 192.168.1.1

# 常见 UDP 端口
sudo rustnmap -sU -p 53,67,68,123,161,162 192.168.1.1

# 带版本检测
sudo rustnmap -sU -sV 192.168.1.1

# 与 TCP 结合
sudo rustnmap -sS -sU 192.168.1.1
```

### 常见 UDP 端口

| 端口 | 服务 | 描述 |
|------|------|------|
| 53 | DNS | 域名系统 |
| 67/68 | DHCP | 动态主机配置 |
| 69 | TFTP | 简单文件传输 |
| 123 | NTP | 网络时间协议 |
| 161/162 | SNMP | 简单网络管理 |
| 500 | ISAKMP | VPN 密钥交换 |
| 514 | Syslog | 系统日志 |
| 520 | RIP | 路由信息 |

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 中等 |
| 速度 | 慢 |
| 可靠性 | 中等 |
| 防火墙友好 | 低 |

### 优点

1. **发现 UDP 服务**：检测 TCP 扫描遗漏的服务
2. **标准方法**：成熟的技术

### 缺点

1. **慢**：由于无响应导致大量超时
2. **模糊结果**：难以区分开放与被过滤
3. **资源密集**：需要发送大量数据包

## TCP FIN 扫描 (`-sF`)

### 描述

FIN 扫描发送仅设置 FIN 标志的数据包。根据 RFC 793，关闭的端口应该回复 RST，而开放的端口应该忽略该数据包。

### 工作原理

```
Scanner          Target
   |    FIN      |
   | ----------> |
   |   (nothing) |  <- 端口开放（按 RFC 793）
   |             |

   |    FIN      |
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |

   |    FIN      |
   | ----------> |
   |   RST       |  <- 端口开放（非 RFC 兼容，如 Windows）
   | <---------- |     或关闭/被过滤
```

### 用法

```bash
# FIN 扫描
sudo rustnmap -sF 192.168.1.1

# 特定端口
sudo rustnmap -sF -p 22,80,443 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 速度 | 快 |
| 最佳用途 | UNIX 系统 |
| Windows 响应 | 全部关闭 |

### 平台差异

| 操作系统 | 行为 |
|----------|------|
| UNIX/Linux | 遵循 RFC 793 |
| Windows | 对所有端口发送 RST |
| Cisco | 遵循 RFC 793 |
| BSD | 遵循 RFC 793 |

## TCP NULL 扫描 (`-sN`)

### 描述

NULL 扫描发送没有设置任何 TCP 标志的数据包。与 FIN 扫描类似，RFC 793 规定关闭的端口应该回复 RST，而开放的端口忽略该数据包。

### 工作原理

```
Scanner          Target
   |   (no flags)|
   | ----------> |
   |   (nothing) |  <- 端口开放
   |             |

   |   (no flags)|
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |
```

### 用法

```bash
# NULL 扫描
sudo rustnmap -sN 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 速度 | 快 |
| 最佳用途 | UNIX 系统 |

## TCP XMAS 扫描 (`-sX`)

### 描述

XMAS 扫描发送设置了 FIN、PSH 和 URG 标志的数据包，"像圣诞树一样点亮数据包"。与其他隐秘扫描类似，它依赖于 RFC 793 的行为。

### 工作原理

```
Scanner          Target
   | FIN+PSH+URG |
   | ----------> |
   |   (nothing) |  <- 端口开放
   |             |

   | FIN+PSH+URG |
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |
```

### 用法

```bash
# XMAS 扫描
sudo rustnmap -sX 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 速度 | 快 |
| 最佳用途 | UNIX 系统 |
| 数据包外观 | 异常 |

---

## TCP ACK 扫描 (`-sA`)

### 描述

ACK 扫描用于映射防火墙规则集，而非确定开放端口。它发送 ACK 数据包并分析端口是被过滤还是未被过滤。

### 工作原理

```
Scanner          Target
   |    ACK      |
   | ----------> |
   |    RST      |  <- 端口未被过滤（无论状态）
   | <---------- |

   |    ACK      |
   | ----------> |
   |   (timeout) |  <- 端口被过滤
   |             |
   |   or ICMP   |
   |  admin-proh |
```

### 用法

```bash
# ACK 扫描用于防火墙映射
sudo rustnmap -sA 192.168.1.1

# 确定防火墙规则
sudo rustnmap -sA -p 1-65535 192.168.1.1
```

### 端口状态

| 响应 | 状态 | 含义 |
|------|------|------|
| RST | 未被过滤 | 端口未被防火墙过滤 |
| 超时/ICMP | 被过滤 | 端口被防火墙过滤 |

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 最佳用途 | 防火墙映射 |
| 确定内容 | 过滤状态 |

---

## TCP Window 扫描 (`-sW`)

### 描述

Window 扫描类似于 ACK 扫描，但检查 RST 响应的 TCP Window 字段来确定端口是开放还是关闭。

### 工作原理

```
Scanner          Target
   |    ACK      |
   | ----------> |
   | RST(Window) |  <- Window > 0：开放
   | <---------- |     Window = 0：关闭
```

### 用法

```bash
# Window 扫描
sudo rustnmap -sW 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 可靠性 | 低（依赖系统） |
| 最佳用途 | 特定系统 |

---

## TCP Maimon 扫描 (`-sM`)

### 描述

Maimon 扫描（以 Uriel Maimon 命名）发送带有 FIN 和 ACK 标志的数据包。某些 BSD 系统在端口开放时会丢弃该数据包（从而暴露端口状态）。

### 工作原理

```
Scanner          Target
   |  FIN+ACK    |
   | ----------> |
   |   (nothing) |  <- 端口开放（某些 BSD）
   |             |

   |  FIN+ACK    |
   | ----------> |
   |    RST      |  <- 端口关闭
   | <---------- |
```

### 用法

```bash
# Maimon 扫描
sudo rustnmap -sM 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 是 |
| 隐秘 | 是 |
| 最佳用途 | BSD 系统 |

## FTP Bounce 扫描 (`-b`)

### 描述

FTP Bounce 攻击利用具有代理功能的 FTP 服务器来反弹扫描。FTP 服务器充当代理，使扫描看起来源自 FTP 服务器。

### 工作原理

```
Scanner        FTP Server      Target
   |   Connect  |              |
   | ---------> |              |
   |   Login    |              |
   | ---------> |              |
   |   PORT     |              |
   | ---------> |              |
   |   PASV     |              |
   | ---------> |              |
   |            |   Connect    |
   |            | ---------->  |  <- FTP 服务器连接到目标
   |            |   Response   |
   |            | <----------  |
   |  Response  |              |
   | <--------- |              |
```

### 用法

```bash
# FTP Bounce 扫描
rustnmap -b ftp.example.com 192.168.1.1

# 使用用户名/密码
rustnmap -b user:pass@ftp.example.com:21 192.168.1.1
```

### 特性

| 特性 | 值 |
|------|-----|
| 需要 root | 否 |
| 隐秘 | 是 |
| 需要 | 易受攻击的 FTP 服务器 |
| 现代使用 | 罕见 |

### 优点

1. **无需 root**
2. **隐藏源地址**

### 缺点

1. **现今罕见**：现代 FTP 服务器不允许此操作
2. **慢**：需要 FTP 协议开销
3. **有限端口**：只能扫描 FTP 服务器可到达的端口

## 扫描类型对比

| 扫描类型 | Root | 隐秘 | 速度 | 可靠性 | 最佳用途 |
|----------|------|------|------|--------|----------|
| `-sS` SYN | 是 | 高 | 快 | 高 | 通用 |
| `-sT` Connect | 否 | 低 | 中 | 高 | 无 root |
| `-sU` UDP | 是 | 中 | 慢 | 中 | UDP 服务 |
| `-sF` FIN | 是 | 高 | 快 | 低 | UNIX 隐秘 |
| `-sN` NULL | 是 | 高 | 快 | 低 | UNIX 隐秘 |
| `-sX` XMAS | 是 | 高 | 快 | 低 | UNIX 隐秘 |
| `-sA` ACK | 是 | 高 | 快 | 中 | 防火墙映射 |
| `-sW` Window | 是 | 高 | 快 | 低 | 高级 |
| `-sM` Maimon | 是 | 高 | 快 | 低 | BSD 系统 |
| `-b` FTP Bounce | 否 | 高 | 慢 | 低 | 遗留系统 |

## 选择正确的扫描

### 快速决策指南

```
你有 root/管理员权限吗？
├── 否 → 使用 -sT（Connect 扫描）
└── 是 → 你的目标是什么？
    ├── 通用端口扫描 → -sS（SYN 扫描）
    ├── UDP 服务 → -sU（UDP 扫描）
    ├── 防火墙规则映射 → -sA（ACK 扫描）
    └── UNIX 上 IDS 规避 → -sF/-sN/-sX
```

### 场景示例

#### 内部网络审计

```bash
# 快速 SYN 扫描带服务检测
sudo rustnmap -sS -sV -T4 -p- 192.168.1.0/24
```

#### 外部渗透测试

```bash
# 带诱饵的隐秘扫描
sudo rustnmap -sS -T2 -f -D RND:10 10.0.0.1
```

#### 防火墙评估

```bash
# ACK 扫描映射规则
sudo rustnmap -sA -p- 192.168.1.1
```

#### 无 Root 权限

```bash
# Connect 扫描
rustnmap -sT -p 22,80,443,8080 target.example.com
```

## 端口状态定义

| 状态 | 含义 |
|------|------|
| `open` | 服务正在监听 |
| `closed` | 端口可访问，无服务 |
| `filtered` | 无法确定状态（防火墙） |
| `unfiltered` | 端口可访问（ACK 扫描） |
| `open|filtered` | 无法确定开放或过滤 |
| `closed|filtered` | 无法确定关闭或过滤 |
