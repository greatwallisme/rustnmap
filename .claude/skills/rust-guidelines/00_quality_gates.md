# Quality Gates

## Zero Tolerance Policy

ALL Rust code must pass with zero errors and zero warnings.

**This is non-negotiable:**
- rustc errors/warnings
- cargo clippy (all, pedantic, nursery, cargo, restriction)
- cargo doc warnings
- cargo fmt issues
- cargo test failures

Any code that does not meet these standards is NOT considered complete.

---

## NEVER Do These (Prohibited Practices)

**1. NEVER use global `#![allow(...)]` attributes:**
```rust,ignore
// FORBIDDEN - this bypasses ALL lints
#![allow(dead_code)]
#![allow(clippy::all)]
```

**2. NEVER modify Cargo.toml to lower standards:**
```toml,ignore
# FORBIDDEN - do NOT disable lints
[lints.rust]
dead_code = "allow"

[lints.clippy]
pedantic = "allow"
```

**3. NEVER comment code instead of deleting.**
Delete unused code. Git preserves history.

**4. NEVER use `unwrap()` or `expect()` for error handling:**
```rust,ignore
// FORBIDDEN
let val = map.get("key").unwrap();
let val = map.get("key").expect("must exist");

// ALLOWED - truly unreachable case with justification
let val = map.get("key").unwrap_or_else(|| unreachable!(
    "key must exist by contract: values are pre-populated in init()"
));
```

**5. NEVER use `todo!()` or `unimplemented!()` in committed code:**
```rust,ignore
// FORBIDDEN in committed code
fn process(&self) -> Result<()> {
    todo!()
}
```

**6. NEVER delete or disable tests to pass checks.**
Fix implementation or test logic, never remove test coverage.

## ALWAYS Do These (Required Standards)

**1. ALWAYS use `?` for error propagation:**
```rust,ignore
// GOOD
fn get_value(map: &HashMap<String, u32>, key: &str) -> Option<&u32> {
    map.get(key)
}

// GOOD - with Result
fn get_value_or_error(map: &HashMap<String, u32>, key: &str) -> Result<&u32, Error> {
    map.get(key).ok_or_else(|| Error::NotFound(key.to_string()))
}
```

**2. ALWAYS use proper error types (`Result`, `Option`):**
```rust,ignore
// GOOD - return Result for fallible operations
pub fn parse_config(path: &Path) -> Result<Config, ParseError> {
    let content = fs::read_to_string(path)?;
    serde_json::from_str(&content).map_err(ParseError::Json)
}

// GOOD - return Option for genuinely optional values
pub fn get_cached(&self, key: &str) -> Option<&Value> {
    self.cache.get(key)
}
```

**3. ALWAYS implement proper iterator patterns:**
```rust,ignore
// BAD - needless collection
let count = items.iter().collect::<Vec<_>>().len();

// GOOD - direct iteration
let count = items.iter().count();

// BAD - manual filter/map
items.iter().filter(|x| x.is_some()).map(|x| x.unwrap())

// GOOD - use flatten
items.iter().flatten()
```

**4. ALWAYS document public APIs with doc comments:**
```rust,ignore
/// Validates the input configuration.
///
/// # Errors
///
/// Returns an error if:
/// - Required fields are missing
/// - Field values fail validation
///
/// # Examples
///
/// ```
/// let config = Config::from_file("config.toml")?;
/// ```
pub fn validate(&self) -> Result<(), ValidationError> {
    // ...
}
```

## When to Use `#[allow(...)]`

**MAY ACCEPT with mandatory justification:**

```rust,ignore
// FFI binding - must match C library symbol names
// See: https://example.org/api.h#L42
#[allow(non_snake_case)]
extern "C" {
    fn ExternalFunction();
}

// Generated by derive macro - upstream issue #123
// https://github.com/example/crate/issues/123
#[expect(clippy::default_trait_access)]
#[derive(Message)]
struct ProtoMessage {}
```

**Rules for `#[allow]` usage:**
1. Use `#[expect]` instead of `#[allow]` when possible
2. Add comment explaining WHY
3. Include reference to upstream issue or specification
4. Keep scope minimal (item-level over module-level)

## Verification Commands

Run these commands before claiming code is complete:

```bash
# 1. Format check
cargo fmt --all -- --check

# 2. Type check
cargo check --workspace --all-targets --all-features

# 3. Clippy - strictest lints
cargo clippy --workspace --all-targets --all-features -- \
  -D warnings -D clippy::all \
  -W clippy::pedantic -W clippy::nursery -W clippy::cargo -W clippy::restriction

# 4. Documentation
RUSTDOCFLAGS="-D warnings" cargo doc --workspace --no-deps --all-features

# 5. Tests
cargo test --workspace --all-targets --all-features
```

**All commands must exit with code 0.**

---

## Fix Priority Decision Framework

When errors/warnings are found, follow this order:

```
1. Improve code      ← Fix the actual code to be correct
2. Refactor design  ← Change the approach if current one cannot be clean
3. Delete unused    ← Remove code that serves no purpose
```

**Do NOT:**
- Bypass with `#[allow]`
- Lower standards in Cargo.toml
- Comment out the problematic code

---

## Workspace Projects

For workspace projects, always use `--workspace` flag:
```bash
cargo fmt --all
cargo check --workspace --all-targets --all-features
cargo clippy --workspace --all-targets --all-features -- -D warnings
cargo test --workspace --all-targets --all-features
```
